# Claude Code プロジェクト設定

このファイルは、このプロジェクトでのClaudeの振る舞いを設定します。

## プロジェクト概要

- **プロジェクト名**: claude_work1
- **目的**: Claude Codeを使用した開発作業の学習と実践
- **言語**: Python, JavaScript, その他

## 作業方針

- コードは日本語コメントで記述
- 小さな変更でもこまめにコミット
- テスト可能なコードを優先
- セキュリティを考慮した実装

## 技術スタック

- Python 3.x
- Node.js（必要に応じて）

## プロジェクト構造

```
claude_work1/
├── CLAUDE.md           # このファイル（プロジェクト設定）
├── MEMORY.md           # メモリインデックス
├── hello.py            # サンプルスクリプト
└── memory/             # メモリ保存先
```

## 開発ガイドライン

1. **コードスタイル**: PEP 8（Python）、ESLint（JavaScript）
2. **コミットメッセージ**: 日本語で変更内容を明確に記述
3. **ドキュメント**: 重要な機能はdocstringやJSDocで記述
4. **エラーハンドリング**: 適切な例外処理を実装

## 禁止事項

- 機密情報のコミット（APIキー、パスワード等）
- ハードコードされた設定値（環境変数を使用）
- 未テストのコードの本番環境へのデプロイ


このプロジェクトでは以下を守ってください。

- 個人開発向けのMVPを優先する
- 過剰な抽象化を避ける
- まず動く最小実装を作る
- 既存ファイルの責務を崩しすぎない
- 変更対象以外はなるべく触らない
- 例外処理はシンプルに、原因が追いやすい形にする
- テストしやすい純粋関数を優先する
- LLM には全文を送らず、短い入力を作る前提で設計する
- 出力フォーマットは JSON か明確な構造に固定する
- 不要な依存追加はしない
